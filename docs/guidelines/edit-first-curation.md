# Edit-first curation

**Rule:** when authoring into a growing corpus (stories, guidelines, schemas, docs), editing an existing artifact is the default. Writing a *new* file is the exception and must be justified: what search terms you ran, what the closest existing artifact was, and why editing it was not the right move.

## Why this matters

Left unchecked, an authoring agent prefers writing new files over finding and extending existing ones, and the corpus bloats with near-duplicates that quietly contradict each other. That has three costs:

- **Split authority.** A consumer faced with two near-duplicate artifacts does not know which to trust, and someone then has to spend energy reconciling them.
- **Orphaned history.** An edit keeps the revision history and the evidence/event chain attached to the original artifact. A new file orphans both — the proof trail starts over.
- **Lost duplication defence.** Search-before-write is the cheapest defence against duplication we have. This rule enforces it by making "I didn't know that already existed" an unacceptable answer.

## What to do

1. **Search first.** Before authoring anything, search the corpus for the concept by its load-bearing terms, not just an exact title. Read the closest hit in full.
2. **Default to edit.** If an existing artifact covers the surface, extend or amend it. This keeps one authority and one history.
3. **Justify any new file.** If you write a new artifact, your return states the search terms run, the closest existing artifact, and why editing it was wrong (genuinely distinct surface, distinct owner, distinct proof shape — not mere convenience).

If you believe the edit-first default itself is wrong for a case, surface a proposal to revise *this* guidance rather than quietly diverging. The default is universal; the exception is earned per artifact.

Composes with [deep-modules](deep-modules.md): edit-first keeps the corpus from fragmenting into shallow near-duplicates; deep-modules gives the deletion-test heuristic for deciding whether two candidate artifacts are genuinely distinct or one is a pass-through of the other.
